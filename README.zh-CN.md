## 这是什么

现代开放世界游戏剧本动辄上百万词，并且常常要跨多种语言交付。体量过了某个临界点，**靠人脑保证设定
不自相矛盾就不再现实**——一处"阵营写反了"会被本地化的语言数放大成 N 轮返工。

OWCopilot 不是"让 AI 替你写游戏"，而是 **帮你核对、检索、组织** 你已经写下的内容：

- **内容图谱**——每个阵营、角色、地点、任务、事件与关系，都以纯文件存储，可在 git 里 diff、可版本化。
- **确定性一致性审计**——能证明"这里引用了不存在的对象""这个前置发生在依赖它的任务之后"，给出
  结构化证据，**全程不用 LLM**。
- **带引用的检索问答**——每个答案都挂着出处；**世界里没讲的，它就明确拒答，绝不编造。**
- **受约束生成**（任务、角色、对话树——或从一句话生成一个接地的**初稿世界作为起点**，而非替代你的创作）
  ——只允许引用图谱内已有的实体，过"生成→评审→精修"环，最后进**待审队列**——人审采纳是 AI 内容进入正典的唯一通道。
- **可编辑的可视化视图**——星图式关系图、编年时间线、底特律式分支对话编辑器、快照差异对比。
  在画布上改动会**直接写回你的文件，与手工编辑走同一条管线**。
- **数据 + 本地化交付**——把世界导成带校验和的 `content_bundle.json`，加本地化 **XLIFF 1.2 / CSV**（UI 字数上限映射为 `maxwidth`）；并支持**引擎导入**，把引擎侧改过的任务行读回人审队列。

它以**单一本地进程**运行（FastAPI 托管 Vue 前端），同时提供做 CI 门禁的 **CLI**、服务化的
**REST API**、以及给 agent 使用的**安全工具面**。

## 我们重点做什么

OWCopilot 围绕一条很窄但实用的闭环设计：**结构化内容入文件 → 确定性校验 → 接地的 AI 建议 →
人工确认 → 审计后交付。** 它帮助团队把大型叙事世界维持在可审阅、可解释、可交付的状态。

| 重点 | OWCopilot 做什么 |
|---|---|
| 结构化正典 | 把设定存成内容图谱，并落在 git 友好的纯文件中。 |
| 确定性质检 | 用结构化证据发现引用断裂、时间线冲突、任务逻辑问题、对话条件作用域问题、本地化风险与注入风险。 |
| 接地式协助 | 问答带引用，证据不足就拒答；生成草稿只能引用已知实体。 |
| 人掌控流程 | AI 草稿与修复建议必须经过审阅、溯源、影子复跑审计、操作者记录与回滚机制。 |
| 交付 | 导出带校验和的数据包与 CSV/XLIFF 本地化文件，并能把引擎侧改过的任务行读回审阅队列。 |

核心纪律保持简单：**确定性校验 → LLM 建议 → 人工确认 → 落盘 → 复跑审计。**
每个对象都带 `origin`（human / ai_draft / ai_patch）与 `review_status`，一条命令即可导出
"AI 参与度"报告。

**它的定位。** OWCopilot 是叙事管线中的一致性、检索、审阅与交付层。适合用于需要进入版本控制的世界设定、
需要在改动前理解影响范围的团队、需要 AI 协助但不能脱离已批准正典的创作流程，以及需要干净数据和标准本地化
文件交付的项目。它叠在既有写作与引擎工作流之上，不试图成为所有创作决策发生的唯一入口。

## 功能特性

| 支柱 | 能力 |
|---|---|
| 🗂️ **内容中枢** | JSON / CSV / XLSX（含中文表头）/ Markdown 多源导入，默认 dry-run。内容即文件；SQLite 只存可重建的运行态。 |
| 📖 **接得住任何输入** | 把文稿、表格、设定稿带入结构化世界。长篇会按覆盖计划拆解，避免静默截断；系统保留原文语言与专名，复查遗漏，并把原文支撑不足的内容标出来。语言、粒度与复查节奏由系统承担。 |
| 🛡️ **一致性审计** | 确定性规则（引用完整性 / 图谱关系 / 世界观语义 / 区域关卡 / **任务逻辑——死锁 / 不可达阶段 / 未定义变量 / 阵营声望引用** / 对话条件作用域 / 本地化预检 / 注入扫描 / AI 信任），结构化证据。审计的是**在 OWCopilot 内编写**的内容（内容即源）——它不是给外部叙事工具文件做 lint 的导入器。baseline 棘轮让存量项目当天接入 CI 门禁。 |
| 🕸️ **影响分析** | 改之前先看波及面：纯图遍历给出"必须改 / 建议查"清单——零成本、零幻觉。 |
| 🔮 **世界问答** | 不只查单条设定，也能回答"主要势力是什么""这几条线如何互相牵制"这类宏观问题。具体问题追到出处，宏观问题先看世界总览；证据不足就说明缺口，而不是补故事。 |
| ⚒️ **修复闭环** | issue → 确定性修复器 + LLM 候选 → **影子副本复跑审计**（会引入新错误的候选直接丢弃）→ 人工应用（记录操作者）→ 一键回滚。 |
| 🎭 **受约束生成** | 分阶段接地创世、扩写、任务、角色、对话树——只引用图谱内实体，过评审精修环，全部进待审队列。创世可**直接以"内容带入"的产物打底**（借鉴灵感库 + 纳入已入档设定），并按原文语言生成。 |
| 🧭 **Agent 协作** | 给人和 agent 同一套受限工具：诊断、检索、影响分析、修复提案与交付检查。Agent 可以调查和提出下一步，但不能绕过审阅把内容写进正典。 |
| ✦ **可视化视图（可编辑）** | 关系星图（拖动 / 连线 / 聚焦 / 涟漪预览）、编年时间线（定序 / 标违例）、对话流编辑器、正典快照 diff。布局确定性计算，编辑走正常管线入正典。 |
| 📦 **数据 + 本地化交付** | 世界导成带校验和的 `content_bundle.json`（任意导入器可读的通用交付）+ 本地化 **CSV 与 XLIFF 1.2**（CAT/TMS 标准；UI 字数上限映射为 `maxwidth`），所有产物附 `sha256` manifest。**引擎导入**把引擎侧改过的任务行回拉进审阅队列。（已去掉"为具体引擎生成代码"：各家引擎 schema 千差万别、无可移植的一键范式，价值在干净数据 + 标准本地化格式。） |
| 💰 **成本工程** | 所有模型调用过唯一网关：双层缓存、级联路由、输出上限、逐操作成本回显与预算护栏。离线默认 **$0**。 |

## 快速开始

```bash
git clone <repo> && cd openworld
python -m venv .venv && .venv/Scripts/pip install -e ".[dev,serve]"   # Windows
# source .venv/bin/activate && pip install -e ".[dev,serve]"          # macOS/Linux
# 追加 ".[semantic]" 启用真·多语言语义检索：首次联网下载一次 bge-m3，之后本地 $0 运行；
# 不装则检索自动降级为 BM25 + 图扩展。

# 一分钟验收：生成内置双语示例世界，跑完整门禁链
.venv/Scripts/python -m owcopilot.cli.main eval-acceptance --workspace .tmp/demo
```

`eval-acceptance` 会生成一个 65 实体 / 10 区域 / 36 任务的双语世界，并验证五道门禁（干净世界零误报、
25 个埋错全检出、影响分析零漏报、30 问句检索全命中、问答拒答纪律）。它**不需要 API Key**——离线
确定性替身让整套验收 **$0** 跑完。

### 启动工作台

```bash
npm --prefix frontend install && npm --prefix frontend run build   # 仅首次
.venv/Scripts/python -m uvicorn owcopilot.service.api:create_app --factory --port 8000
# 浏览器打开 http://localhost:8000 —— 单进程、零配置；世界存在 ~/.owcopilot/worlds/
```

侧边栏按工序分区：**概览**（世界总览 / 设定档案）·**创世·创作**（创世 / 人物 / 创作：任务·对话·台词·物案）·
**内容带入**（文稿提炼 / 表格导入 / 灵感库）·**校勘·分析**（校勘修复 / 影响分析 / 专项清查 /
**时间线** / **关系星图**）·**问答·交付**（世界问答 / 审阅台 / 导出）·**管理**（工作区 / **变更史** / 设置）。
首次打开有「新手引导」逐区讲解。

**首次启动是空的**——不内置示例世界。两种方式上手：把你已有的内容*带入*（文稿 / CSV / XLSX / Markdown，
在「内容带入」），或从一句话*生成*（在「创世·创作」，需先接入模型）。空的「世界总览」会引导你去「管理·世界」新建或打开一个。

**接入你自己的模型：** 打开「设置」选择服务商（DeepSeek / OpenAI / Anthropic / Kimi / 智谱 / 通义 /
豆包 / 自定义），粘贴 API Key、选模型、测试连接。**Key 只进本机进程内存、调用直连服务商，没有任何
中间服务器**。不接入模型也能翻档案、审阅、导出。

## 工作方式

```
  已有内容 / 新设想 / 引擎回流
              ↓
  内容中枢：结构化、版本化、可审阅
              ↓
  质量层：一致性审计 · 影响分析 · 时间线 · 任务逻辑 · 本地化预检
              ↓
  协作层：世界问答 · Agent 诊断 · 修复提案 · 接地草稿
              ↓
  人审层：审阅队列 · 操作者记录 · 影子复跑 · 回滚
              ↓
  交付层：数据包 · 本地化文件 · 引擎侧改动回收
```

**技术栈：** Python 3.11+、FastAPI、Pydantic v2、NetworkX、SQLite(WAL)；Vue 3 + Vite + TypeScript
（**零图形库依赖**——可视化视图全是手写、确定性 SVG）；OpenAI 兼容的 LLM 网关。离线确定性替身让
整套测试 **$0** 运行。

## 四种使用形态

- **Web 工作台**——主路径；单进程，FastAPI 托管 Vue。
- **CLI**——`audit`（CI 门禁，有未解决 error 即非零退出）、`impact`、`suggest`/`apply`/`rollback`、
  `ask`、`draft`/`expand`、`export`、`eval-acceptance`。每条命令输出 JSON。
- **REST API**——按项目资源化，长任务走 SSE；`OWCOPILOT_API_KEY` 守护花钱的 `llm_mode=real`
  调用（非本机客户端 fail-closed）。
- **MCP 服务**——面向 agent 生态的安全工具面；可诊断、检索、提出修复和检查交付，但不暴露直接写正典动作。

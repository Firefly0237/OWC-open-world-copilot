## 真问题

### 【中】审阅台「AI 评审准确度」的核心解释段落，样式被一对中文引号打挂了
- 一句话：审阅台那段「什么是漏检、数字怎么看」的关键说明文字，因为 `class` 属性用了中文全角引号 `”`，CSS 完全没生效，会以超大默认字号 + 满色渲染，跟旁边的小字说明明显不一致，看起来像页面坏了。
- 根因（非表象）：`frontend/src/pages/ReviewPage.vue:402`
  ```
  <p class=”muted ct-desc”>
  ```
  用的是 U+201D 全角引号 `”` 而非 ASCII `"`。Vue 模板解析器把它当成 `class=”muted` 和 `ct-desc”` 两个畸形/布尔属性，于是 `.muted`（小字+暗色）和 `.ct-desc`（0.8rem）样式都没应用到这段。
  - 已用 python 确认该行含 `”`(U+201D)、不含 `class="`。
  - 紧邻的下一段 `.ct-thin`（样本不足提示）样式正常，对比之下这段会突兀地大一号、亮一截。
- 证据：`frontend/src/pages/ReviewPage.vue:402`；`git blame` 显示是 **R1（commit 2e10f59「Round 1 fixes」）引入的**——讽刺的是，这正是当轮要打磨「AI 评审准确度」文案的改动，却把这段说明的样式写挂了，R2 没发现，残留至今。
- 为什么没被挡住：前端构建只有 `vue-tsc --noEmit && vite build`（见 `frontend/package.json`），没有会校验模板 HTML 属性引号的 ESLint；Vue 解析器对畸形 `class` 容错不报错，所以两轮都没暴露。
- 真 bug 还是设计选择：**真 bug**。不是闸门能兜底的东西——它就是给非技术审阅者看的解释文案，偏偏样式坏在最该清楚的那一段。
- 修复：把 `”...”` 改回 `"muted ct-desc"` 即可（一处）。

### 【低-中】「lint」这个程序员黑话漏到了策划眼前（物案/对话树两处），同页台词那处却已是人话
- 一句话：创作工坊里，生成物品/技能/成就文案后会显示「X 条被 lint 拦下」，生成对话树会显示「X lint」——策划根本不知道「lint」是啥；可同一页「台词」那栏却写的是人话「X 条因引用越界被拦下」，三处不统一。
- 根因：`frontend/src/pages/CreationPage.vue`
  - `:366` 物案：`{{ flavorResult.rejected }} 条被 lint 拦下。`
  - `:282` 对话树：`{{ dlgResult.lint }} lint`
  - 对照 `:326` 台词：`{{ barkResult.rejected }} 条因引用越界被拦下。` ← 这才是策划看得懂的写法
- 证据：上面三行。`git blame` 显示「lint」两处早在 `22719ee`(2026-06-17) 就在，**早于 R1/R2，两轮都没改到**，属残余。
- 真 bug 还是设计选择：**面向用户文案的真问题（轻）**。功能本身没搭壳（确实在按规则过滤），只是把内部规则名「lint」直接吐给了不懂技术的人，且同页已有更好的措辞没沿用。建议统一成「N 条未通过校验/因不合规被拦下」之类，并把那条命中规则用人话说清。
